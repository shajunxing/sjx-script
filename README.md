# SJX Script，纯C实现的类JS语法的极简引擎，Version 2.0

下载地址 <https://github.com/shajunxing/sjx-script/releases> ，包含 windows 和 linux 可执行程序。

创建文本文件 hello.js ，内容为 `let name = input("What's your name? "); print("Greetings", name);` ，执行 `js hello.js` ，输入名字并回车，就能看到欢迎语。

源代码由多行语句组成，首行可以是 shebang 格式，即 `#!` 开头的，遵循 posix 约定。

大部分语句以分号 `;` 结尾。

## 变量和值

标识符以字母或 `_` 开头，中间只能包含字母、数字或 `_` 。

变量以标识符命名，变量定义语句格式为 `let 变量名;` ，赋值语句格式为 `变量名 = 值;` ，变量可以在定义时候赋值 `let 变量名 = 值` ，如果不赋值，默认为 `null` 。

数值类型包括：

- 空值，字面量为 `null` ，表示不存在;
- 布尔值，字面量为 `true` 和 `false` ；
- 数字，字面量同c语言浮点型，引擎内部存储为 double ；
- 字符串，字面量类似c语言字符串字面量，约定 utf-8 编码，支持 `\' \" \? \\ \a \b \e \f \n \r \t \v \0` ， `\e` 表示 0x1b ，即ansi转义符；
- 数组，用 `[]` 括起来的0个或多个数值，之间用 `,` 分隔；
- 对象，用 `{}` 括起来的0个或多个键值对，之间用 `,` 分隔，键和值用 `:` 分隔，键是字符串或简写为标识符，但内部都解析为字符串，值是任意数值；
- 函数，字面量表示为 `function (参数) { 函数体 }` 。（不支持`=>`表达式，因为观感不佳容易与等号混淆）

数组用 `数组[数字]` 访问其中成员，数字是下标，是从 0 开始的正整数，0 是第一个成员。

对象用 `对象[字符串]` 或 `对象.标识符` 访问其中成员，还可用 `对象?.标识符` ，当对象为 `null` 时，返回 `null` 。

数组和对象读取不存在的成员等同于读取赋值为 `null` 的成员，都是返回 `null` ，不做区分。

函数赋值给变量 `let 名字 = function(){};` 可以写成 `function 名字(){}` ，结尾不需要分号。

## 表达式和运算符

大部分运算符功能类似c语言。运算符的优先级从低到高为：

- 三元运算符 `?` `:`
- 逻辑或运算符 `||`
- 逻辑与运算符 `&&`
- 关系运算符 `==` `!=` `<` `<=` `>` `>=`
- 加减运算符 `+` `-`
- 乘除运算符 `*` `/` `%`
- 指数运算符 `**`
- 前缀运算符 `+` `-` `!` `typeof`
- 数组/对象成员访问和函数调用运算符 `[]` `.` `?.` `()` `::`

运算符遵循严格规则，没有隐式转换。只有布尔值可以进行逻辑运算。

三元运算符 `条件 ? 表达式1 : 表达式2` ，当条件为 `true` 时表达式2不会执行， `false` 时表达式1不会执行。

如果有多个串联的逻辑或表达式，那么当计算到第一个 `true` 之后的表达式不会执行。

如果有多个串联的逻辑与表达式，那么当计算到第一个 `false` 之后的表达式不会执行。

`== !=` 判断两个变量相等的规则是：

- 如果类型不同，那么不等；
- 如果都是 `null` ，那么相等；
- 如果都是布尔值，那么同为 `true` 或同为 `false` 时候相等；
- 如果都是数字，那么做浮点值比较；
- 如果都是字符串，那么用 c 的 strcmp 比较；
- 如果都是其它类型，那么只有指向同一个值时候才相等，而非比较内容。

数字支持所有关系和数值运算符。字符串支持所有关系运算符和连接操作 `+` ，用 c 的 strcmp 比较大小。

`typeof` 返回数值/表达式的字符串表示的类型，分别是 `null boolean number` `string` `array` `object` `function`。

赋值表达式 `=` `+=` `-=` `*=` `**=` `/=` `%=` `&&=` `||=` `++` `--` 类似 c 语言，但不返回值。

展开语法 `...数组` 将数组所有成员拎出来，由另一个数组或作为参数由另一个函数接收，例如 `let a = [1, 2, 3]` ，那么 `[...a, 4]` 相当于 `[1, 2, 3, 4]` ， `foo(0, ...a)` 相当于 `foo(0, 1, 2, 3)` 。

## 语句

块语句 `{}` 自身是语句，内部可以包含多个语句。

条件语句是 `if (条件) 语句1 else 语句2` ， `else` 可选。

循环语句有三类： `while (条件) 语句` ； `do 语句 while (条件)` 和 `for` 循环，条件必须是布尔值。

`for` 循环支持以下语法，`[]` 表示可选部分：

- `for ([[let] 变量名 = 表达式 ] ; [条件] ; [赋值表达式])`
- `for ([let] 变量名 in 数组/对象)`
- `for ([let] 变量名 of 数组/对象)`

`for in` 循环数组下标或对象键， `for of` 循环数组或对象的值，它们只循环非 `null` 成员。

`break;` 语句退出循环， `continue;` 语句跳转到循环开头。

`return 表达式;` 语句如果在函数内部，意为函数返回值；如果在函数外部，则是退出程序并返回值。程序默认返回 `null` ，如果返回值不是 `null` ，脚本解释器一般会打印它。

运行时错误处理语句 `try {语句组1} catch (变量名) {语句组2}` ，错误是一个数值，可以用 `throw 表达式;` 手动抛出。如果语句组1里出现错误，那么会赋给 `catch` 的变量名，然后执行语句组2。 `catch` 是可选的，如果没有，则不处理错误。如果没有错误处理语句，那么错误会发给解释器，一般会终止程序并打印错误。

（不支持`finally`，因为我认为根本不需要，反而会使代码执行顺序显得怪异）

（其实可以简化为 `try 语句1 catch (变量名) 语句2` ，如果单个语句可以不用大括号，但不兼容javascript，其实主要是因为vscode不支持缩进，否则 `try 语句;` 还是很漂亮的）

## 变量作用域

作用域就是可见范围，不可见也就是不存在。

变量对于源代码里出现在它定义之前的语句都是不存在的，访问会引发未找到错误。所以变量作用域只存在与定义之后。

在大括号里面的变量，对于外面的语句而言也是不存在的。所以变量作用域都是局部的。函数参数与函数体处于相同作用域。

这些都符合编程语言的约定俗成。

## 函数

函数总是会返回值，未写 `return` 语句的，默认返回 `null` 。

函数支持默认参数 `参数名 = 表达式` ，如果没有对应的实参也没有默认参数，那么为 `null` 。

函数也支持剩余参数 `...参数名`，剩余参数必须在最后，会把未收集的所有实参按顺序合并为数组，例如函数 `foo(a, b, ...c){}` ，调用 `foo(1, 2, 3, 4)` ，那么 `c` 为 `[3, 4]`。

语法糖 `数值::foo(参数)` 相当于 `foo(数值, 参数)` ，据此可写出优雅的链式语句，比如 `arr::filter()::map()::reduce()`

函数支持闭包。在定义时，也就是创建函数数值的时候，该函数依照作用域约定把当前所能见到的所有变量都记录进函数数值内部，称之为闭包，之后闭包就固定不再变动了。在运行时，函数体内的代码搜索变量是沿着调用栈由栈顶开始逆向到该函数闭包为止。例如：

```
function invoke(func) {
    func();
}
let a = 1;
invoke(function() {
    let b = 2;
    invoke(function() {
        let c = 3;
        invoke(function() {
            print(a, b, c); // 正确显示 1 2 3
        });
    });
});
```

例二：

```
function invoke(func) {
    func();
}
let a = 1;
function outer() {
    print(a, b, c);
}
invoke(function() {
    let b = 2;
    invoke(function() {
        let c = 3;
        invoke(outer); // outer()在定义时不可见b和c，虽然调用outer的代码处于它们的作用域，但outer依旧见不到它们
    });
});
```

例三：

```
function foo() {
    let val;
    function bar(v) {
        val = v;
    }
    function qux() {
        return val;
    }
    return {bar : bar, qux : qux};
}
let f = foo(); // foo()返回bar和qux两个函数数值，变量val存在于bar和qux的闭包里
f.bar(3); // 虽然该代码在foo外部，但还是能访问其内部的变量val
print(f.qux()); // 可以看到val的值被成功修改
```

## 垃圾回收

引擎自动执行引用计数算法。为提高性能，未使用的变量和数值会被缓存而不是被释放，以加速下一次申请。

如果存在循环引用，引用计数算法是无法处理的。比如 `let a = [], b = []; a[0] = b; b[0] = a;` ，可以 `a[0] = null;` 打破循环。再比如 `let a; let b = function() {}; a = function() {};` ，`a` 和 `b` 都存在对方的闭包里面，形成循环引用。

引擎内置标记清除算法 `gc()` 函数，该函数同时释放所有缓存。标记清除算法是很耗时的，所以留给程序自行决定是否执行。一般而言，编码时候应该尽量避免循环引用。

## 内置函数库

命名规则：

- 最常用的，取用最常用的名称，例如控制台输入和输出 `input` `print`；
- 单一特征的，匹配单个 windows / posix 命令或 c 标准库 / unistd 函数的，例如 `cd`、`md` 和 `rd`，使用最简短的，同时也可以提高执行速度；
- 非单一特征的，自定义名称。

参数和返回值记法：

- -: 空值，无返回值的函数实际上返回空值
- b: 布尔值
- n: 数字
- s: 字符串
- (): 函数
- []: 数组，或可选参数
- {}: 对象
- *: 任意类型
- /: 多种类型或名字分隔符
- ...: 不限参数

一般地，如果函数参数数量少于定义或参数类型与定义不符，会抛出错误。函数执行期间遇到错误也会抛出。
